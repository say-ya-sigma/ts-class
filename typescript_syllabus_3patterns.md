# TypeScript 授業シラバス（3パターン・統一版）

各パターンとも2時間×13回の構成。実質的な学習内容は全パターン共通で、使用する例題・実例のみを対象者に合わせて差別化しています。

---

## 共通学習カリキュラム

| 回 | 実質的な学習内容（全パターン共通） |
|----|----------------------------------|
| 1 | TypeScriptの概要。JavaScriptとの違い、なぜ型が必要か。TypeScriptの歴史と現状。開発環境の準備と初めての実行。 |
| 2 | Node.js/npmの基礎。パッケージ管理とモジュールシステム。tsconfig.jsonの基本設定。ビルドツールの概要。 |
| 3 | プリミティブ型（string/number/boolean）。型アノテーションの基本。型推論の仕組み。変数と関数への型付け。 |
| 4 | 配列型とオブジェクト型。複雑なデータ構造への型付け。readonly修飾子。型エイリアス（type）の導入。 |
| 5 | Union型（A \| B）の概念。Union型を使った型定義。リテラル型による値の制限。Union型のメリットとユースケース。 |
| 6 | Type Narrowing（型の絞り込み）。typeof/in演算子による絞り込み。null/undefinedチェック。型ガード関数の基礎。 |
| 7 | Discriminated Unionパターン。タグ付きUnionによる型の区別。ステートマシン的な型設計。Exhaustive Check。 |
| 8 | 関数の型。コールバック関数の型付け。アロー関数の型。関数型の変数宣言。戻り値の型推論。 |
| 9 | Genericsの基礎。<T>による型パラメータ。汎用関数の作成。extendsによる型制約。配列操作のGenerics関数。 |
| 10 | Utility Typesの活用。Partial/Required/Pick/Omit/Readonly。既存の型から新しい型を生成する方法。 |
| 11 | モジュールシステム。ESModulesのimport/export。名前付きエクスポートとデフォルトエクスポートの使い分け。名前空間。 |
| 12 | 実践演習①。これまでの知識を活用した課題。型安全なコードの設計と実装。エラーハンドリングの実践。 |
| 13 | 実践演習②＋総まとめ。複雑な課題への取り組み。次のステップ（フレームワーク・バックエンド）への橋渡し。 |

## パターンA：2年Webデザイナー向け

**例題の特徴：** 色、画像、Canvas、フォーム、UIコンポーネントなど、視覚的・グラフィカルな題材を使用

| 回 | 学習内容 | 使用する例題・実例 |
|----|----------|-------------------|
| 1 | TypeScript概要・開発環境構築 | 画像ギャラリーの紹介。「色のHEXコードを文字列として管理する」必要性から型の概念を導入。10日で作られたJavaScriptの問題点をアニメーション例で説明。 |
| 2 | Node.js/npm・プロジェクト構築 | ポートフォリオサイトの雛形作成。Viteを使った画像付きWebページのセットアップ。ブラウザで動くJavaScriptとNode.jsで動くJavaScriptの違いを確認。package.jsonで画像圧縮ライブラリなどを管理。 |
| 3 | プリミティブ型・型アノテーション | 色コード（`#FF5733`）をstring型で管理。ピクセル値（1920px）をnumber型で管理。表示/非表示をboolean型で管理。VSCodeのホバーで色プレビュー表示。 |
| 4 | 配列型・オブジェクト型 | 色の配列`string[]`でカラーパレット管理。フォント設定オブジェクト`{ name: string; size: number; weight: string }`。画像メタデータの型付け。 |
| 5 | Union型・リテラル型 | ボタンサイズ`"small" \| "medium" \| "large"`。テーマカラー`"light" \| "dark" \| "auto"`。位置指定`"top" \| "bottom" \| "left" \| "right"`。 |
| 6 | Type Narrowing | Canvasのクリックイベント（MouseEvent \| TouchEventの絞り込み）。画像の読み込み状態チェック。要素の存在確認による型絞り込み。 |
| 7 | Discriminated Union | UIコンポーネントの状態管理。`{ type: "loading" } \| { type: "loaded"; image: HTMLImageElement } \| { type: "error"; message: string }`。画像ローダーの実装。 |
| 8 | 関数の型 | 色の変換関数（hex→rgb）。アニメーションのイージング関数。クリックハンドラー（イベントの型）。画像リサイズ関数。 |
| 9 | Generics | `first<T>`で画像配列の先頭取得。`pick<T>`で画像メタデータから特定プロパティ抽出。色の配列・フォントの配列で共通処理。 |
| 10 | Utility Types | `Pick<ImageData, "width" \| "height">`でサムネイル情報生成。`Partial<Theme>`でテーマの部分的上書き。`Readonly<ColorPalette>`で定数管理。 |
| 11 | モジュールシステム | カラーパレット定義を別ファイル化。ユーティリティ関数（色変換・サイズ計算）をモジュール分割。コンポーネントごとの型定義ファイル。フロントエンドコードとビルド時/サーバー側コードの配置の違いも整理。 |
| 12 | 実践演習① | 画像ギャラリーアプリ（型安全版）。画像URLの配列管理、クリックイベント処理、表示状態の管理をTypeScriptで実装。 |
| 13 | 実践演習②＋総まとめ | フォーム付きポートフォリオセクションの実装。入力値の型チェック、バリデーション、エラーハンドリング。CSR/SSRの違いと、HTML表示後にJavaScriptが有効化されるハイドレーションの初歩を紹介。次ステップ（Reactコンポーネント）の紹介。 |

## パターンB：3年Webデザイナー向け

**例題の特徴：** コンポーネント設計、デザインシステム、React/Next.jsへの橋渡しを意識した題材

| 回 | 学習内容 | 使用する例題・実例 |
|----|----------|-------------------|
| 1 | TypeScript概要・開発環境構築 | コンポーネント設計の観点から型の重要性を説明。AltJS戦争の歴史とTSの勝利理由。Next.jsプロジェクトのTypeScriptセットアップ。 |
| 2 | Node.js/npm・プロジェクト構築 | デザインシステムプロジェクトの構築。StorybookやViteを使ったコンポーネント開発環境。ブラウザ実行とNode.js実行の役割分担、ビルドツールが何をしているかを確認。npmでUIライブラリを管理。 |
| 3 | プリミティブ型・型アノテーション | コンポーネントPropsの基本。`title: string`、`count: number`、`isVisible: boolean`。型推論を活かしたProps設計。 |
| 4 | 配列型・オブジェクト型 | コンポーネントリストの型`Component[]`。Propsオブジェクトの型`{ variant: string; size: string; disabled: boolean }`。メニュー項目の型定義。 |
| 5 | Union型・リテラル型 | バリアント型`"primary" \| "secondary" \| "ghost"`。サイズ型`"sm" \| "md" \| "lg" \| "xl"`。ステータス型`"idle" \| "loading" \| "success" \| "error"`。 |
| 6 | Type Narrowing | フォーム入力値の検証（string \| number \| nullの絞り込み）。APIレスポンスの型チェック。コンポーネントの条件付きレンダリングの型安全化。 |
| 7 | Discriminated Union | 非同期データ取得の状態管理。`{ status: "idle" } \| { status: "loading" } \| { status: "success"; data: T } \| { status: "error"; error: Error }`。SSR後の画面状態とクライアント側で再開される状態管理の違いにも触れる。 |
| 8 | 関数の型 | イベントハンドラー（onClick/onChange/onSubmit）の型。コールバックPropsの型付け。カスタムフック相当の関数の型設計。 |
| 9 | Generics | `SelectProps<T>`で選択肢コンポーネントの汎用化。`ListProps<T>`でリスト表示コンポーネント。`Response<T>`でAPIレスポンスの型付け。 |
| 10 | Utility Types | `Pick<User, "name" \| "email">`でユーザー表示用データ型。`Omit<APIUser, "password">`で公開用データ型。`Partial<Props>`でオプショナルProps。 |
| 11 | モジュールシステム | コンポーネントごとの型定義（Button/types.ts）。エクスポートパターン（named export推奨）。型のみのimport（`import type`）。クライアントコンポーネントとサーバー側処理の責務分離も整理。 |
| 12 | 実践演習① | デザインシステムの型定義。カラーパレット、タイポグラフィ、スペーシングの型を作成。テーマ切り替え機能の型安全な実装。 |
| 13 | 実践演習②＋総まとめ | 既存デザイン（Figmaからのコンポーネント）のProps型設計。React/Next.jsへの型の橋渡し。CSR/SSR/SSGの違い、SSR後のハイドレーション、実務での型設計ベストプラクティスを整理。 |

| 11 | モジュールシステム | レイヤー別のモジュール分割（controller/service/repository）。型定義ファイル（.d.ts）の作成。@typesエコシステムの利用。 |
| 12 | 実践演習① | TODO管理CLIツールの作成。コマンドライン引数の型付け、ファイル入出力、エラーハンドリング。型安全なデータ永続化。 |
| 13 | 実践演習②＋総まとめ | REST APIクライアントの型安全な実装。Zodを使ったランタイムバリデーションと型の統合。次ステップ（Hono/Drizzle等）への橋渡し。 |

## 授業準備のポイント

1. **スライド資料は共通**: 概念説明スライドは全パターンで共通化可能。Webデザイナー向けには「ブラウザ / サーバー / API / HTML / JS」の関係図を追加すると理解しやすい
2. **コード例のみ差し替え**: 各回のデモコードと演習問題を対象者に合わせて用意。3年Webデザイナー向けはCSR/SSR/ハイドレーションのミニデモを含めると効果的
3. **ハンズオン環境**: 
   - 2年Webデザイナー: HTML/CSS + TypeScript（Vite）
   - 3年Webデザイナー: コンポーネント開発環境（Vite + Storybook推奨）
   - 2年システムエンジニア: Node.js CLI環境またはシンプルなAPIモック
